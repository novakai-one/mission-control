Playbook Deltas

The six steps and the named-item gate are proven — keep both exactly as run. These bolt on; each traces to a win or gap above.

Paste, never paraphrase — now as law

The named-row checklist is proven tech: paste it verbatim at every hop. The one legal edit is ADDING rows — the audit did, keep that. Any hop that must shorten it says “shortened” out loud. The gradient stays inverted.

The chief's floor: one record per view — or say the sample out loud

If the independent pass is time-boxed, the close-out reads “drove 4 of 8 (Log, KR, Requests, Learnings); builder rows cover the rest” — never “clicked it all.” A sample honestly named is rigor; a sample rounded up is or002 again.

Drive the numbers like the pixels

Durations in any close-out come from wire timestamps (kickoff → done), not self-report; hypothesis verdicts recompute from the stamps. The chief already holds the wire path — it's pasted in its own heartbeat prompt.

Name the click method in the gate

One more gate column: HOW the click was made. “Real pointer” is the default; eval is allowed only where pointer is impossible, and the row says so. Keep the builder's effect-evidence pattern (record the inspector id per click) — it's excellent.

Adopt the event wake-up before round 3

Already written in or002's playbook, not yet adopted, 19 minutes paid since: builder done/questions go out as --interrupt to the chief; the cron survives only as a silence detector. ~19 min back per hour-scale round.

Commit by name after gate rounds

In a repo with known stale artifacts, git add -A is banned; name the files — or put main.cjs in .gitignore first, which closes this class permanently.

Mint ids that can't collide

Store ids carry the author in the slug (log_2026-07-20-031-chief) or the writer greps the store before minting. One id, one body, forever.

Keep the cold census in the METHOD

This review's ground-truth move — an independent, real-pointer, one-record-per-view census against the merged build, run cold before reading anyone's claims — took ~12 minutes and settled held-vs-compressed beyond argument. Make it a standing step of every operational review. 12 minutes to certainty.

How This Was Traced

Same operation as or002 — dual-format tracing (Kimi wire via wire-tracer.py with an env-overridable BASE; the builder's Claude-format session JSONL parsed directly; the tunnel journal as the handoff clock), run entirely on scratchpad copies because the subject session was still alive. Two techniques new this round, both worth keeping. The claimed-vs-measured diff: take any number a report asserts (a duration, a coverage count), grep every store it landed in, then recompute it from wire timestamps — that is how “~3.25 h” fell apart against 45.2 minutes (and then resolved, in the acceptance review, into the builder's session age: 3.23 h since its 10:09 spawn), and how “clicked it all” resolved to 4 of 8. The cold census: before reading anyone's evidence, drive the merged build yourself with real pointer clicks — one record per view, screenshot what opens — so the run's own screenshots can't anchor what you expect to see. The census is what turns a verdict about process into a fact about the product.
