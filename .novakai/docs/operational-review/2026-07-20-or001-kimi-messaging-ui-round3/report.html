The Playbook For The Next Long Run

Your chief-delegation-method doc (written yesterday) already has the right spine — onboard, plan-then-stop, cross-provider audit, build with boundaries, verify lightly, close out. These eight bolt onto it; each traces to a specific failure above.

Spec states, not screens

Ten minutes before green-light: for every interactive control in scope, one line per state — open, closed, mid-motion, hover, empty. The storyboard can't show a collapsed rail? Then the plan says "collapsed = framed 44px strip, wordmark glyph, reopen chevron in the tab row." The toggle miss was unspecifiable-not-checkable; this makes it both.

Write the taste laws into the repo

A one-page docs/design-laws.md — frame-everything, amber scarcity, 700ms structural motion, calm density, show-don't-tell — linked from every UI brief. Builders invent wherever specs run out; make their default invention your taste instead of generic-app taste.

Make fidelity mechanical

Every UI milestone's evidence includes one composite image: built crop beside reference crop. The gate brief names each comparison explicitly ("M2 rail header vs scene-01 crop") — this run proved the checklist eclipses the prose above it. Assertion from memory doesn't count; a side-by-side file does.

Batch ceremony; keep the checks

One shared isolated stack for the whole run, owned by the chief. Browser sweeps at two or three seams instead of per-item rounds. Keep per-commit tsc/tests — they're 30 seconds and caught real bugs. Target: cut ~200 of 626 steps ≈ 1.5h at observed latency.

Parallelize the disjoint middle

Chief maps milestones to files up front; disjoint ones run as simultaneous builders in separate worktrees, chief merges. This run's graph: M1/M2 → M3 → {M4, M5, M6, M7} → M8 → M9. Same work, ~4.6h → ~2.5h wall.

Checkpoint the owner in pixels

After every milestone push: one screenshot + one-line delta + "k of 9 · ETA" into #team. Costs the chief 60 seconds; gives you veto power from bed; kills the 02:31 black box. Show, don't tell — the app's own doctrine, applied to building the app.

The finish line drives, it doesn't view

Final inspection = click every control at the owner's viewport, capture motion as frame series, re-open the reference scenes fresh in the same context, and only then issue the verdict. If the finish-line agent can't click it, it isn't done.

Pin the infra in the brief

Demo servers: no timeout, stdin detached, from the first launch. Say playwright-core because that's what exists. Allowlist a scratch directory so probes never touch the worktree. All three cost steps this run; all three are one line in the next brief.

How This Was Traced — Reusable

Kimi keeps a per-agent wire.jsonl — every prompt, thought, tool call, result, and per-step token/latency record. 4.7 MB of wire across 9 agents for this run (13 MB with screenshot blobs). I built a small tracer (wire-tracer.py, ~150 lines, lives beside this report) with three verbs, and they answer the three questions any long run raises:

QuestionVerbWhat it does
What happened, when?digest <agent>Collapses an agent's wire into a readable timeline — prompts, thinking, tool calls with compact args, result previews, todo snapshots, token counts per step. 8 hours reads in ~5 minutes per agent.
Where did time and tokens go?statsPer-agent wall-clock, LLM-latency share, first-token medians, tool mix, token totals. This is what exposed the 95%-latency fact and the 20s step tax.
Where did requirement X die?grep <regex>Cross-agent search over every event. Trace any requirement through plan → briefs → tool calls → reports and find the hop where it disappears. This is what convicted the M9 brief, and what showed which builders ever opened a design image.

That third move — chain-of-custody tracing — is the general audit technique for delegation pipelines: every hop that rewrites instructions is a place a requirement can silently die, so grep the requirement's key phrase at each layer's artifact and mark where it stops appearing. It took minutes to run and produced the headline finding of this review.
