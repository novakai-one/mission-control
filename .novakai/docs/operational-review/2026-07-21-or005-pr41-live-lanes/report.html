Case Study · When Manual Cleanup Became Green

The first meaningful weakening occurred in the acceptance layer, not in Chris's prompt or onboarding: a manually overcome teardown failure was treated as evidence of operator cleanup rather than evidence that the shipped cleanup path was incomplete.

Root cause class: Verification (primary); Execution and Management (contributing).

The Contract requires start/stop/restart isolation (brief.md:13-16), and the approved plan explicitly requires “Stop dev (SIGINT) … restart dev” (plan.md:214-223). The implementation's cleanup sends SIGTERM only to the Vite or tsx listener (tools/dev-lane.mjs:51-60), while npm run dev runs those children under concurrently --restart-tries -1 (package.json:11-12). The old supervisor can therefore recreate the listener immediately.

What The Window Proved
SEVERE · Verification
An observed product failure was converted into a passing check

This was not a speculative edge case:

  • Worker reported two teardown leaks caused by respawning concurrently supervisors (result.md:83-84) and recorded that killing listeners is not enough in its session memory.
  • Manager reproduced the respawn race during independent teardown (msg_f2fb4e50-626c-4ecd-9db4-9ae273f47171).
  • Chief reproduced it again at 09:35–09:36; the same supervisors survived SIGTERM, spawned fresh children, and required SIGKILL.
  • The unresolved PR P2 at 09:37:57 independently described the same defect at tools/dev-lane.mjs:59.

Yet result.md marks all checks passed, and C-OWNERSHIP proves only that a foreign listener is left untouched (c-ownership-demo.txt); it never proves that a stale owned supervisor tree is reclaimed. The Chief then reported “Nothing broken, nothing blocked.” At scale, this is dangerous: a team can manually work around a defect three times and accidentally count the workaround as proof of the product behavior.

Exact Prompt Changes Proposed

No onboarding change is warranted. Onboarding was one of the strongest parts of the run.

Chief acceptance — add the contradiction brake

Before (CHIEF.md, Part 5):
“Check every named acceptance item.”

After:
“Check every named acceptance item against all observed evidence. Any failure seen by the Worker, Manager, Chief, CI, or PR review automatically reopens the related check. A manual cleanup or workaround is not a pass. It may become a follow-up only after the Manager shows that it is outside the Mission Contract. Before ACCEPT, reconcile every reported FAIL, respawn, forced kill, manual cleanup, and deviation in result.md.”

Build authorization — require one coherent plan

Add before “Plan approved. Build is authorized”:
“Canonical-state gate: re-open plan.md after all audit and Chief rulings. The plan and its completion checklist must state one consistent decision; messages do not amend the plan. Search for and remove superseded acceptance wording before GO.”

Chief report — make counts and routing claims source-derived

Add before the Part 6 report template:
“Re-open audit.md and result.md immediately before reporting. Copy finding counts exactly. For routing claims, use message envelopes as truth and label terminal-display anomalies as system behavior. If the summary conflicts with primary evidence, correct the summary before sending it.”

Smallest Useful Process Change

Add only the contradiction brake in Chief acceptance: one observed failure reopens its named check until fixed or explicitly proven out of scope. It needs no new artifact, meeting, role, or review stage. This single rule would have stopped the false green here while preserving the speed and warmth that worked.

For gate receipts, have the existing command tee raw output into the existing evidence directory; do not add another ceremony or document.

Scale Decision

Repeat once; do not scale the handbook yet.

Patch the contradiction brake, then run one more substantial managed mission. The next Process Reviewer should check only three things: contradictory evidence reopened a check, plan.md remained internally consistent after rulings, and the final report copied measured counts exactly. If those hold, promote and scale. The delegation skeleton has earned that second run; the acceptance gate has not yet earned autopilot.
